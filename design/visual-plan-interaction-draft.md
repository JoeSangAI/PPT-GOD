# PPT GOD - Visual Plan 交互设计稿

## 一、整体流程

```
Content Plan 确定
    │
    ▼
┌─────────────────────────────────────────┐
│  Step 1: 素材询问（Agent 对话）          │
│  Agent 问：有参考模板/图片/Logo吗？       │
│  用户：上传 or 回答 or 都没有            │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  Step 2: 风格提案（3选1）                │
│  AI 出 3 套风格方案（名称+配色+氛围+字体）│
│  用户挑选 1 套 or 要求调整               │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  Step 3: 模板图提取（如用户上传了PPT）    │
│  后台解析 → AI 推荐封面/目录/内容/封底    │
│  用户可逐类更换                          │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  Step 4: 生成 Visual Plan（每页画面描述） │
│  全局预览页：显示标题/正文/参考图/种子标记 │
│  单页编辑：修改 visual_description       │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  Step 5: 用户确认 Visual Plan            │
│  文字描述 OK → 推进到 Prompt             │
│  不满意 → 回退修改自然语言描述            │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  Step 6: 生成 Visual Prompt（后端自动）   │
│  前端不展示 Prompt，作为生图原料          │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  Step 7: 打样                            │
│  默认勾选 4 类种子页 + 用户额外勾选        │
│  生成图片预览                            │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  Step 8: 种子页确认                        │
│  用户确认种子页 OK → 成为正式参考         │
│  不满意 → 单页重试 or 回退到 Visual Plan  │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  Step 9: 全量生成                         │
│  所有页面以对应种子页为参考批量生图       │
│  生成 PPTX 下载                           │
└─────────────────────────────────────────┘
```

## 三、关键交互说明

### 3.1 配色色块交互

- **常态**：显示圆形或圆角色块（直径 24px）
- **悬浮**：显示 Tooltip，内容为 HEX 值（如 `#1E3A5F`）+ 颜色名称（如"深海蓝"，如有）
- **点击**：无操作（仅展示，不可编辑；编辑需回退到风格提案阶段）

### 3.2 种子页标记交互

- **全局页卡片**：推荐种子页显示 🌱 图标（右上角）
- **悬浮**：Tooltip 显示"种子页：Bookend 家族"
- **点击图标**：弹出菜单，提供"更换种子页"和"取消种子页"选项
- **更换**：弹出候选列表（同 Family 的其他页面缩略图），用户点选替换

### 3.3 模板图"换一张"交互

- **点击"换一张 ↻"**：在同类型页面中找下一个候选
- **动画**：缩略图淡出 → 新缩略图淡入（300ms）
- **循环**：如果所有候选都换过一轮，回到第一个
- **状态标记**：用户手动换过的模板图显示"用户指定"标签，不再被 AI 自动更换覆盖

### 3.4 回退交互

| 用户场景 | 操作路径 |
|---------|---------|
| 单页生图效果不对 | 单页卡片 → 重试按钮（重新生成该页） |
| 某类种子页整体风格不对 | 方案 C 面板 → 种子页状态 → 点击该类种子页 → 编辑 visual_description → 保存 → 重新打样 |
| 整体风格方向错了 | Agent 面板输入"风格太冷了，变暖一点" → Agent 修改配色和氛围 → 回退到 Step 2 重新提案 |
| 布局不满意 | Agent 面板输入"内容页改成上下结构" → Agent 修改 template 推荐 → 重新提取模板图 |

### 3.5 进程指示器交互

- **已完成步骤**：绿色 ✓，可点击回看该阶段的产出
- **当前步骤**：蓝色高亮，显示进行中的状态
- **未开始步骤**：灰色，不可点击
- **点击已完成步骤**：展开该阶段的摘要信息（如点击"Content Plan ✅"显示页数和核心主题）

## 四、图片类型在数据库中的定义

```
ReferenceImage 表
├── id
├── project_id          # 所属项目
├── slide_id            # 所属单页（null = 全局素材）
├── role                # 图片类型
│   ├── template        # 模板图（从用户PPT提取）
│   ├── style_ref       # 风格参考图（官网/海报等）
│   ├── logo            # 品牌Logo
│   ├── content_ref     # 单页内容参考图
│   └── chart_ref       # 数据图表截图
├── file_path           # 文件路径
├── description         # AI 生成的描述（如颜色提取结果）
└── created_at
```

## 五、前端状态机

```
draft（草稿）
  │
  ▼ Agent 引导生成 Content Plan
planning（Content Plan 编辑）
  │
  ▼ 用户点击"生成 Visual Plan"
visual_planning（Visual Director 询问素材 → 风格提案 → 模板提取）
  │
  ▼ 用户确认风格，生成每页 Visual Plan
visual_ready（Visual Plan 预览/编辑）
  │
  ▼ 用户点击"生成 Prompts"
prompt_ready（Prompt 已生成，前端不展示）
  │
  ▼ 用户点击"打样"
prototype（打样中，显示种子页生成进度）
  │
  ▼ 用户确认种子页
prototype_ready（种子页确认，可全量生成）
  │
  ▼ 用户点击"全量生成"
generating（全量生图中）
  │
  ▼ 全部完成
completed（可下载 PPTX）
```

## 六、与现有代码的对应关系

| 交互稿中的概念 | 现有代码中的字段/文件 |
|--------------|-------------------|
| Content Plan | `Slide.content_json` |
| Visual Plan | `Slide.visual_json`（已有 page_num/type/layout/seed_family/visual_description/design_notes） |
| Visual Prompt | `Slide.prompt_text` |
| 种子页标记 | `Slide.visual_json.seed_family` + 新增 `is_seed_recommended` / `is_seed_confirmed` |
| 风格提案 | 新增 `Project.style_proposal`（JSON 存储 3 套方案） |
| 配色方案 | 风格模板 `templates/styles/{style_id}.md` 中的 palette |
| 模板图 | 新增 `ReferenceImage.role = "template"` |
| 风格图 | 已有 `ReferenceImage.role = "style_ref"` |
| Logo | 已有 `ReferenceImage.role = "logo"` |
| 参考图 | 已有 `ReferenceImage.role = "content_ref"` |
| 图表图 | 已有 `ReferenceImage.role = "chart_ref"` |
