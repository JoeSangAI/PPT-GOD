# 视觉素材交互重构计划

## 当前问题诊断

1. **上传后不可见**：用户点了"上传参考图"，只弹出一个 Toast"参考图已上传"，但界面上没有任何地方显示这张图的缩略图、文件名、类型。用户不知道传成功了没有。
2. **没有素材管理**：已上传的素材不能查看、不能删除、不能替换。用户传错了只能干瞪眼。
3. **Agent 与上传脱节**：Agent 在聊天里问"有没有素材"，用户在 UI 上传了，但 Agent 不知道，聊天流里没有反馈。
4. **复合上传逻辑模糊**：同时上传 Logo + 参考图 + 模板，系统怎么综合处理？用户不知道。
5. **素材与风格提案的关联不清晰**：上传了素材后，风格提案会自动更新吗？用户不知道。

## 设计原则

- **立即可见**：上传成功后，素材必须立刻出现在用户视野中
- **可管理**：每个素材都能删除、替换
- **状态透明**：用户清楚知道"已上传什么素材"、"这些素材会怎样影响结果"
- **流程闭环**：上传 → 展示 → 分析 → 反馈 → 生成

## 方案设计

### 一、新增「设计素材面板」（核心组件）

在视觉阶段的主舞台中，增加一个固定位置的素材展示面板。

**位置**：主舞台顶部（内容规划 slides 上方，或风格提案区域上方）

**布局**：
```
🎨 设计素材
┌────────────┬────────────┬────────────┬─────────────────────┐
│ 🎯 Logo    │ 📎 参考图1 │ 📎 参考图2 │ 📑 模板 (4页)       │
│ [缩略图]   │ [缩略图]   │ [缩略图]   │ [封面缩略图]        │
│ [删除×]    │ [删除×]    │ [删除×]    │ [展开查看] [删除×]  │
└────────────┴────────────┴────────────┴─────────────────────┘
          [+ 上传素材]          [没有素材，直接开始提案 →]
```

**素材类型展示规则**：
| 类型 | 显示方式 | 最多数量 |
|------|---------|---------|
| Logo | 单张缩略图，带"🎯 Logo"标签 | 1张（新上传替换旧的） |
| 参考图 | 横向排列缩略图，带"📎 参考"标签 | 不限制（支持多张） |
| 模板 | 显示"📑 模板 (X页)"，点击展开查看每页缩略图 | 1套（新上传替换旧的） |

**删除交互**：
- 每个素材卡片右上角有「×」删除按钮
- 删除后前端立即移除，调用后端 API 删除文件和数据库记录
- 删除后如果风格提案已生成，提示"已删除素材，是否重新生成风格提案？"

### 二、上传流程优化

**当前流程**：点击按钮 → 选文件 → 上传 → Toast 提示 → 结束

**优化后流程**：
1. 点击"上传参考图/Logo/模板"按钮
2. 选文件
3. 上传中：按钮显示「上传中...」
4. 上传成功：
   - 素材面板立即显示新上传的素材缩略图
   - 显示 Toast「✅ Logo 已添加」
   - 如果风格提案已生成，按钮旁出现提示「已上传新素材，点击重新生成提案更新风格」
5. 上传失败：按钮恢复，显示 Toast「❌ 上传失败：原因」

**复合上传场景**：
- 用户可以同时上传多个参考图（多次点击上传，每次一张）
- Logo 只能有一张：新上传自动替换旧的（前端提示「已替换旧 Logo」）
- 模板只能有一套：新上传自动替换旧的（前端提示「已替换旧模板」）

### 三、素材与风格提案的联动

**状态流转**：
```
用户上传素材
  ↓
素材面板显示新素材
  ↓
如果风格提案已存在 → 提示"是否基于新素材重新生成？"
  ↓
用户点击"重新生成" → 后端分析所有素材 → 更新风格提案
  ↓
主舞台风格提案卡片刷新
```

**后端分析逻辑**：
- 生成/重新生成风格提案时，自动分析当前项目的所有素材
- Logo → 提取品牌色、调性
- 参考图 → 提取配色、风格、字体
- 模板 → 标记为"有模板"，影响布局推荐
- 多参考图 → 取第一张分析，或未来支持多张综合分析

### 四、Agent 聊天与素材上传的联动

**当前问题**：Agent 在聊天里问素材，用户在 UI 上传了，但聊天流里没有痕迹。

**优化方案**：

方案 A（推荐）：上传后自动在聊天流中插入一条系统消息
```
用户：[文字消息]
🤖 系统：用户已上传 Logo（文件名.png）
Agent：收到，我会基于你的 Logo 提取品牌色和调性，在风格提案中体现。
```

方案 B：上传后前端显示一个"告知 Agent"按钮，用户点击后发送

**推荐方案 A**，因为用户操作越少越好。

### 五、前端数据流

**新增/复用状态**：
- 复用 `project.reference_images`（后端已有）
- 复用 `project.selected_template_recommendations`（后端已有）
- 前端从 `selectedProject` 中派生素材列表

**素材面板的渲染逻辑**：
```typescript
const visualAssets = useMemo(() => {
  const logo = project.reference_images?.find(r => r.role === "logo");
  const refs = project.reference_images?.filter(r => r.role === "style_ref") || [];
  const hasTemplate = project.selected_template_recommendations != null;
  return { logo, references: refs, hasTemplate };
}, [project]);
```

### 六、后端接口

**已有接口**：
- `POST /upload` — 上传文件（role: style_ref/logo/template）
- `POST /extract-template` — 上传模板并提取
- `DELETE /reference-images/{id}` — 删除参考图（需要确认是否有）

**需要新增/确认**：
- `DELETE /reference-images/{id}` — 删除单条参考图记录和文件
- 模板删除：需要清空 `selected_template_recommendations` 和 `reference_images` 中 role=template 的记录
