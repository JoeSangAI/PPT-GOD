# PPT God ✨

### 让 AI 做出一套真正能交付的 PPT

> 把主题、文档、逐字稿、Logo、产品截图、参考图和修改意见，变成一套风格一致、品牌准确、产品可信的演示文稿。

PPT God 不是一个只会“快速生成几页”的 PPT 工具。它更像一位懂内容、懂视觉、懂品牌交付的 PPT 搭档：先帮你想清楚每页讲什么，再帮你统一整套视觉，最后生成可以继续修改和交付的 PPT 文件。

---

## 🌱 品牌故事

PPT God 最早来自一次真实分享任务：我需要为一个行业论坛准备 PPT。

我试过很多 AI 工具。NotebookLM 很会理解资料，Lovart 很会提供视觉灵感，普通 AI PPT 工具也能快速出稿。但真正要交付时，问题不再是“能不能生成一页好看的图”，而是：

- Logo 不能被 AI 重新画错。
- 页面不能各自好看，却放在一起不像一套作品。
- 产品截图、界面、实物和关键卖点不能被自由发挥。
- 每一页讲什么，必须先让人看得懂、改得动、确认得了。

所以 PPT God 的起点不是“更快地出一版草稿”，而是让 AI 按照真实 PPT 的标准工作：理解内容，尊重素材，统一视觉，交付一份真正能用的文件。

## ⭐ 核心特性

### 🧠 1. 从粗糙想法到每页内容

你不需要一开始就准备好完整大纲。PPT God 可以从主题、长文档、逐字稿、报告或零散想法出发，整理出一套可讲述的页面结构。

它会先帮你拆出每页标题、正文重点和页面角色，让你在生成前知道：这套 PPT 到底讲什么、每一页承担什么作用、哪里需要补充或调整。

**适合：**

- 把一篇长文档变成演示稿
- 把一次分享逐字稿变成课件
- 把粗略想法整理成正式提案
- 把研究材料压缩成汇报 deck

---

### 🛡️ 2. Logo 保真

很多 AI 工具会把 Logo 当作画面元素重新生成，结果字形、比例、颜色都可能出错。商业 PPT 里，这不是小问题。

PPT God 把 Logo 当成品牌资产处理：它应该稳定、清晰、准确地出现在需要品牌露出的页面里，而不是被模型随意改写。

**解决的问题：**

- 客户 Logo 不能变形
- 品牌色不能漂移
- Logo 位置不能遮挡主体
- 同一套 PPT 里 Logo 表现要一致

---

### 🧬 3. 跨页面一致

一页好看，不代表一套 PPT 好用。正式演示需要整体感：封面、章节页、内容页、数据页、产品页和结束页都要属于同一个视觉系统。

PPT God 会围绕整套演示建立统一方向，让不同页面既有变化，又不会变成一堆互不相关的海报。

**你会得到：**

- 统一的视觉气质
- 稳定的页面节奏
- 一致的品牌呈现
- 更像“成套作品”的最终 PPT

---

### 📸 4. 产品展示准确

如果 PPT 里有 App 截图、SaaS 界面、硬件照片、产品结构图或真实业务画面，准确性比“看起来高级”更重要。

PPT God 会把产品素材作为事实依据，帮助你围绕真实产品做表达，减少“看起来像，但其实不对”的问题。

**适合：**

- 产品发布会
- SaaS 方案介绍
- App 功能展示
- 硬件或实物产品介绍
- 客户案例和业务场景说明

---

### 🎨 5. 参考图和模板驱动视觉方向

你可以上传参考图、旧 PPT、PDF 页面或品牌素材，让 PPT God 理解你想要的视觉方向。

它不是简单套模板，而是把这些素材转化成整套 PPT 的风格依据：色彩、质感、版式气质、页面层级和品牌氛围都可以被吸收。

**适合：**

- 沿用客户已有品牌风格
- 参考一套成熟提案的视觉语言
- 把旧材料升级成更完整的新提案
- 用少量参考图确定整套 deck 的审美方向

---

### 🧭 6. 生成前先判断

普通 AI PPT 常见的问题是：一键生成很快，但生成后才发现每页讲什么都不对。

PPT God 更强调先判断再生成。你可以先看到每页内容、页面角色和视觉方向，再决定是否继续生成。这能减少返工，也让你更容易把控最终结果。

**你可以提前确认：**

- 每页标题是否准确
- 内容重点是否完整
- 页面顺序是否合理
- 视觉方向是否符合预期
- 哪些页面需要重点打磨

---

### 💬 7. 修改意见尽量落到页面上

PPT God 的目标不是只在聊天里回复“好的，我明白了”。当你说：

- “第 3 页更商务一点”
- “这页突出产品截图”
- “整体更像发布会”
- “把内容压缩得更适合路演”
- “Logo 不要挡住主体”

这些意见应该尽量反映到具体页面和最终 PPT 里。

---

### 📦 8. 导出可继续修改的 PPT

PPT God 不是只给你一组灵感图。最终目标是一份可以下载、检查、修改和交付的 PPT 文件。

这意味着它更适合真实工作场景：你可以先让 AI 生成高质量初稿，再在 PowerPoint、WPS 或 Keynote 里继续做最后调整。

## 🚀 怎么使用？

1. **输入内容**：给它一个主题、一份文档、一段逐字稿或一组想法。
2. **上传素材**：加入 Logo、产品截图、参考图、PDF 或 PPT 模板。
3. **确认内容**：先看每页讲什么，调整标题、正文和结构。
4. **选择方向**：确认整体视觉风格，让整套 PPT 保持一致。
5. **生成页面**：生成可预览的页面效果。
6. **继续修改**：用自然语言调整某一页或整套风格。
7. **下载 PPT**：得到可以继续编辑和交付的文件。

## 🆚 和普通 AI PPT 工具有何不同？

| 普通 AI PPT 工具 | PPT God |
| --- | --- |
| 快速出一版草稿 | 面向正式交付 |
| 更像模板填充 | 更关注内容判断和视觉统一 |
| 单页看起来不错 | 整套页面保持一致 |
| Logo 容易被画错 | Logo 作为品牌资产保真 |
| 产品图容易被自由发挥 | 产品素材作为事实依据 |
| 生成前很难判断内容 | 先确认内容与方向，再生成 |
| 聊天回答和页面修改脱节 | 修改意见尽量落到具体页面 |
| 长材料容易变成机械拆页 | 更关注讲述结构和页面取舍 |

---

## 🧪 为什么这些特性重要？

因为正式 PPT 的难点，不是把文字塞进模板，而是同时守住四件事：

- **内容判断**：哪些内容该讲，哪些内容该删，顺序如何组织。
- **视觉秩序**：几十页看起来像同一套作品。
- **品牌准确性**：Logo、品牌色、素材不能随便漂移。
- **产品可信度**：截图、界面、实物和关键卖点不能被编造。

PPT God 想做的，就是让 AI 从“帮你想一想”走向“帮你做出一份真正能用的 PPT”。

## 🤖 Agent CLI：更新已有项目的内容规划

外部 Agent 可以用严格 Markdown 新建项目，也可以在原项目内同步内容。已有项目默认只预览差异，不会直接写入：

先检查本地服务、Agent 合同、当前账号和项目连接：

```bash
python scripts/pptgod_cli.py doctor
python scripts/pptgod_cli.py capabilities
python scripts/pptgod_cli.py whoami
python scripts/pptgod_cli.py list-projects
```

```bash
python scripts/pptgod_cli.py update-content-plan <project_id> path/to/plan.md
```

确认 `changed`、`added`、`deleted`、`unchanged` 和删除风险后，再明确应用：

```bash
python scripts/pptgod_cli.py update-content-plan <project_id> path/to/plan.md --apply --open
```

同步会尽量保留已匹配页面的稳定 ID、素材绑定、画面方案、Prompt、图片和历史版本。它不会自动确认内容、切换项目阶段或启动视觉生成。完整格式和工作流见 [`docs/agent/content-planning-playbook.md`](docs/agent/content-planning-playbook.md)。

正文同步会同时更新富文本编辑器使用的 `content_blocks` 与 Markdown 正文镜像，并在提交前读回验证；应用成功时响应中的 `readback.ok` 必须为 `true`。本地整合版默认打开 `http://localhost:8000`，5173 端口仅用于前端开发模式。

启动视觉方案、画面 Prompt 或图片生成后，可以让 CLI 等待持久化 run 到达终态，避免 Agent 自己写轮询脚本：

```bash
python scripts/pptgod_cli.py wait <project_id> --run-id <run_id>
```
