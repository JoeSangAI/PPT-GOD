# PPT God

### 让 AI PPT 从“能生成”，走向“敢交付”

把主题、文档、逐字稿、Logo、产品图和参考图，变成一套风格一致、内容可控、品牌准确、可以继续修改的 PPT。

## 先看结果：不同内容，也应该拥有不同的视觉系统

PPT God 不把所有内容塞进同一个模板。它会先理解这次演示要解决什么问题，再决定整套 PPT 应该长什么样。

![PPT God 品牌提案与课程课件公开案例](./docs/readme-assets/case-variety.jpg?v=20260719-wow)

左边是品牌与商业提案，右边是课程与知识分享。内容任务、视觉语言和页面节奏完全不同，但每一套内部都保持连续。

## AI PPT 最大的问题，不是不会生成，而是不敢直接交付

正式 PPT 需要同时守住内容、视觉、品牌和交付。任何一项失控，都会让“已经生成”变成“还得重做”。

| 普通 AI PPT | PPT God |
| --- | --- |
| 快速生成几页 | 面向一整套正式交付 |
| 每页各自好看 | 跨页保持同一视觉系统 |
| Logo、产品和图表容易被自由发挥 | 关键素材作为事实依据 |
| 生成以后才发现内容不对 | 生成前逐页确认内容与画面 |
| 得到一组灵感图 | 导出可以继续修改的 PPTX |

**成套，不是拼图 · 可控，不是盲盒 · 准确，不是重画 · 可交付，不是只供欣赏**

## 8 页像一套作品，而不是 8 张随机海报

封面、金句、数据、场景、培训、医疗教育和结束页承担不同任务，但色彩、字体、构图气质与品牌露出共同组成一个视觉系统。

![Apple Vision Pro 八页跨页一致性展示](./docs/readme-assets/apple-deck-grid.jpg)

## 关键素材不能被 AI 自由发挥

原始 Logo、原始产品图和最终页面放在一起，才能看清“保留了什么、融合了什么”。Logo 保持原样，产品主体高还原，版式、空间和光影则为演示重新设计。

![原始 Logo 与产品图生成最终 PPT 的对照](./docs/readme-assets/apple-brand-proof.jpg)

准确和美观不是二选一。对于品牌发布、产品介绍和客户提案，关键事实必须准确，其他画面才有创作空间。

## 生成不是盲盒：每一页都能看懂、选准、改动

批量生成之前，先确认每页标题、正文、页面类型、参考素材和视觉方向。需要修改时，可以选中单页、多页或整套 PPT 继续调整。

![PPT God 页面级内容与画面控制](./docs/readme-assets/page-control.jpg?v=20260719-native)

## 从材料到可交付 PPTX，PPT God 怎样工作

1. **输入材料**：主题、Markdown、文档、逐字稿、旧 PPT、PDF 都可以成为起点。
2. **确认每页内容**：先看标题、正文重点、页面角色和讲述顺序。
3. **选择视觉方向**：比较不同风格提案，确定这一套 PPT 的视觉系统。
4. **决定素材如何使用**：Logo、图表、产品、人物和氛围图采用不同处理方式。
5. **生成、微调并导出**：逐页检查，继续修改，最后导出可编辑 PPTX。

## 风格可以选择，参考素材也有正确的使用方式

### 同一份内容，先看风格提案，再决定往哪做

PPT God 不把“风格”锁死在一个模板里。同一份内容可以先比较不同方向，再选择真正适合这次任务的一套视觉语言。

![同一份内容的两套风格提案](./docs/readme-assets/style-choice.jpg)

### 参考图有三种命运：保留、融合、精修

![精确粘贴、智能融合与精修融合对照](./docs/readme-assets/asset-routing.jpg)

- **精确粘贴｜事实不改**：Logo、二维码、图表、UI 截图和原始证据保持原样，并在落版时自动避让文字。
- **智能融合｜氛围融入**：场景、人物和氛围图进入整体构图、光影与视觉风格。
- **精修融合｜融合 + 高还原**：先融入画面，再校准主体边缘、比例和关键细节，适合复杂产品与高还原要求。

## 适合哪些工作

| 使用场景 | 你交给 PPT God 什么 | 最终解决什么问题 |
| --- | --- | --- |
| 品牌 / 产品发布 | Logo、产品图、发布内容 | 品牌准确，产品可信，整套像发布会 |
| 商务 / 研究汇报 | 报告、数据、方案材料 | 内容先理清，重点更适合演示 |
| 课程 / 知识分享 | 长文档、逐字稿、知识资料 | 从机械拆页变成有讲述节奏的课件 |
| 创意 / 生活方式 | 想法、参考图、审美方向 | 先比较风格，再生成完整视觉系统 |

## 案例驱动，让下一次默认更稳

真实案例不只是作品，也是 PPT God 的训练场。每一次交付中的反馈，都会先脱敏为回归案例，再沉淀成可复用的能力与质量检查。

**真实反馈 → 脱敏回归 → 能力升级 → 下一次默认更稳**

<details>
<summary><strong>Agent / CLI 接入</strong></summary>

外部 Agent 可以先检查本地服务、账号和项目连接：

```bash
python scripts/pptgod_cli.py doctor
python scripts/pptgod_cli.py capabilities
python scripts/pptgod_cli.py whoami
python scripts/pptgod_cli.py list-projects
```

更新已有项目的内容规划时，默认只预览差异；确认后再应用：

```bash
python scripts/pptgod_cli.py update-content-plan <project_id> path/to/plan.md
python scripts/pptgod_cli.py update-content-plan <project_id> path/to/plan.md --apply --open
```

完整格式和工作流见 [Agent 内容规划说明](./docs/agent/content-planning-playbook.md)。

</details>
